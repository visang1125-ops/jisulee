# 백엔드 배포 대안

## 🎯 선택지

Railway 외에도 여러 옵션이 있습니다:

## 1. Render (추천 대안)

### 장점
- ✅ 완전 무료 플랜 (제한적이지만)
- ✅ 사용하기 쉬움
- ✅ GitHub 연동
- ✅ 자동 배포

### 배포 방법
1. [render.com](https://render.com) 가입
2. "New +" → "Web Service"
3. GitHub 저장소 연결
4. 설정:
   - Build Command: `npm run build`
   - Start Command: `npm start`
5. 배포 완료

### 무료 플랜 제한
- 서비스가 15분 동안 요청이 없으면 sleep (깨어나는데 시간 소요)
- 소규모 프로젝트에 적합

## 2. Fly.io

### 장점
- ✅ 무료 플랜 제공
- ✅ 빠른 배포
- ✅ 전 세계 CDN

### 배포 방법
```bash
# Fly CLI 설치
npm i -g flyctl

# 로그인
flyctl auth login

# 배포
flyctl launch
```

## 3. 로컬에서만 실행 (개발용)

### 장점
- ✅ 완전 무료
- ✅ 빠른 개발
- ✅ 제한 없음

### 단점
- ❌ 외부에서 접근 불가
- ❌ 항상 컴퓨터가 켜져 있어야 함
- ❌ 프로덕션에는 부적합

### 사용 방법
```bash
npm run dev
```

## 4. 같은 Vercel에 배포 (서버리스 함수)

### 장점
- ✅ Vercel 하나로 통합
- ✅ 무료 플랜
- ✅ 간단한 설정

### 단점
- ❌ 파일 시스템 제한 (Excel 파일 읽기 어려움)
- ❌ 코드 수정 필요 (서버리스 함수로 변환)

### 필요 작업
- Express 앱을 서버리스 함수로 변환
- 파일 시스템 대신 데이터베이스 사용

## 5. 다른 클라우드 서비스

### AWS, Google Cloud, Azure
- 무료 티어 제공
- 설정이 복잡할 수 있음
- 대규모 프로젝트에 적합

## 💡 추천

### 프로덕션 배포
1. **Railway** - 가장 쉬움, 무료 크레딧 제공
2. **Render** - 완전 무료, sleep 제한 있음

### 개발/테스트
- **로컬 실행** - 가장 빠르고 간단

### 빠른 시작
- **Render** - 설정이 간단하고 무료

## 🚀 Render 배포 (간단 버전)

1. [render.com](https://render.com) 가입
2. "New +" → "Web Service"
3. GitHub 저장소 연결: `visang1125-ops/jisulee`
4. 설정:
   - Name: `jisulee-backend`
   - Environment: `Node`
   - Build Command: `npm run build`
   - Start Command: `npm start`
5. "Create Web Service" 클릭
6. 배포 완료 후 URL 확인

## ❓ 어떤 것을 선택해야 할까?

### Railway 선택하는 경우
- ✅ 가장 쉬운 설정
- ✅ 안정적인 서비스
- ✅ 무료 크레딧으로 시작

### Render 선택하는 경우
- ✅ 완전 무료
- ✅ 간단한 설정
- ✅ sleep 제한이 괜찮은 경우

### 로컬 실행 선택하는 경우
- ✅ 개발/테스트만 필요
- ✅ 외부 접근 불필요
- ✅ 빠른 개발
